## 3. デザインパターン

**初回実装**: Pattern 1 を採用する。

### Pattern 1: 近未来SF
- コンセプト: 暗いベースにネオンのシアン/ティールが浮かぶ、計測機器のUIを想起させる表現
- 背景: ディープネイビーのグラデーション + 薄いグリッド/スキャンライン
- アクセント: レーダーの外周・数値・アクティブ要素に発光（glow）
- カード: ガラス調（半透明 + ぼかし + 細いボーダー）
- アイコン/ロゴ: 単色化 or 発光縁取りで統一
- モーション:
  - 画面表示時にレーダーチャートがフェードイン + 拡大
  - KPI値は0→目標値へカウントアップ
  - 詳細テーブルのセクションは段階的に出現

**カラーパレット（例）**
- Base: #050B14
- Surface: #0B1626
- Accent: #3BE3F6
- Accent-2: #22B9D2
- Text: #E6F4FF
- Muted: #8BA3B8
- Warning: #F5C542

**タイポグラフィ（例）**
- Heading: "Oxanium" or "Orbitron"
- Body: "Zen Kaku Gothic New" or "Rajdhani"

### Pattern 2: ブランドブルー基調
- コンセプト: 明るいニュートラル背景にブランドブルーを主役とした、クリーンで信頼感のあるUI

- アクセント: ブルートーンを主軸に、状態色でセカンダリアクセントを付与

## 4. 画面定義

### SCR-01: レポート概略（ダッシュボード）

**目的**
- 選手の主要KPIを一目で把握し、全体評価と次の確認ポイントを提示する

**対象ロール**
- 参加者（選手）
- 指導者（閲覧権限がある範囲）

**レイアウト構成**
- ヘッダー: サービスロゴ + 画面タイトル
- 選手セレクタ: 画面上部に選手名ドロップダウン（CSVの選手名から抽出）
- 選手セレクタの見出しテキストはメインブルーで強調
- KPIサマリー: レーダーチャート（8軸） + 各軸のスコア/順位
- Score Snapshot: 8カテゴリのスコア/順位カード（2列×4行）
- フッター: 連携サービスロゴ行（Photon/Hawkin を表示。左右均等に配置し視覚バランスを取る。その他は将来拡張で非表示）
- ハイライト動画カード: サムネイル + 再生ボタン + ラベル

**表示項目**
- 選手名（ドロップダウンで切替）
- カテゴリKPI（8領域）
  - スピード
  - 加速
  - 減速
  - 再加速
  - ジャンプ
  - パワー
  - 安定性
  - バランス・左右差
- 各カテゴリ表示要素
  - スコア（70-95）
  - 順位（例: 1位/2位/3位）
- ハイライト動画
  - サムネイル
  - 再生URL

**UI表記（主要ラベル）**
- 選手
- 選手を選択
- KPIサマリー
- スコアサマリー
- 測定日
- レポート詳細
- ハイライト動画

**操作**
- 選手セレクタ変更で表示内容を切替
- レポート詳細ボタン/レーダーチャートのタップで SCR-02 へ
- 動画カードのタップで再生

**状態**
- 取得前: スケルトン表示
- データ欠損: 該当KPIは「--」表示
- 将来拡張カテゴリ（方向転換/疲労度/メンタル等）は非表示
- 権限不足: KPI値は非表示、ダミーの枠のみ

---

### SCR-02: レポート詳細

**目的**
- KPIの内訳を「前回/今回」で比較し、改善点を具体化する

**対象ロール**
- 参加者（選手）
- 指導者（閲覧権限がある範囲）

**レイアウト構成**
- ヘッダー: ロゴ + 画面タイトル
- 比較テーブル: 左から「項目名 / 前回 / 今回」の3列
- SPは項目カード内で「前回/今回」を2列表示（見出し行は非表示）
- セクションごとに区切り、提供元アイコンを表示

**セクション/項目（例）**
- スプリント（Photon）
  - 総合タイム
  - 5m 到達タイム
  - 10m 到達タイム
  - 15m 到達タイム
  - 20m 到達タイム
  - 最大速度
  - 加速度
- アジリティ 5-0-5（Photon）
  - 同一カード内で「加速/減速/再加速」の小見出しを付けて区分する
  - 加速
    - 平均加速度
    - 進入5mタイム
    - 最大速度
  - 減速
    - 平均減速度
  - 再加速
    - 平均再加速度
    - 再加速5mタイム
- ジャンプ（Hawkin CMJ）
  - 跳躍高
  - ジャンプモメンタム
  - 沈み込み量
  - 滞空時間
  - 離地時間
- パワー（Hawkin CMJ）
  - 推進最大パワー
  - 推進最大パワー (体重比)
  - 制動最大パワー
  - 制動最大パワー (体重比)
- 安定性（Hawkin CMJ）
  - 着地最大力
  - 着地最大力 (体重比)
- バランス・左右差（Hawkin CMJ）
  - 左右差: 制動最大力
  - 左右差: 推進最大力
  - 左右差: 着地最大力
  - 左右優位表示（左が優位/右が優位/同等、左=ブルー・右=レッドで強調）

**UI表記（主要ラベル）**
- レポート詳細
- 項目
- 前回
- 今回
- タップで折りたたみ

**操作**
- 戻るで SCR-01 へ
- セクション見出しタップで折りたたみ
- 項目タップで説明/指標の評価方法のモーダルを表示

**状態**
- 取得前: スケルトン表示
- データ欠損: 前回/今回の欠損を「--」表示
- オフライン: キャッシュ済みデータを優先表示
- 将来拡張カテゴリ（方向転換/疲労度/メンタル等）は非表示

### SCR-03: 監督用ダッシュボード（/coach）

**目的**
- チーム内の状態を一覧し、主要KPIの差分を素早く把握する（1/18測定データを基準）

**対象ロール**
- 指導者（コーチ/監督）

**レイアウト構成**
- ヘッダー: 画面タイトル + 戻る導線
- フィルタ/ソート: 1行目=ソート（選手名/学年/ポジション/背番号）、2行目=絞り込み（学年/通期）
- 左カラム: 選手リスト（背番号/名前/ポジション/学年）
- 右上: 選手セレクタ情報（名前/背番号/ポジション/測定日）
- 右中: レーダーチャート（8軸）+ スコア + 差分ラベル
- 右下: 選択選手のサマリー（総合/主要カテゴリのスコア + 差分）

**表示項目**
- フィルタ/ソート
  - ソート: 選手名/学年/ポジション/背番号
  - 絞り込み: 学年（全学年/高3/高2/高1）、期間（通期/上半期/下半期）
- 選手リスト
  - 背番号、名前、ポジション、学年（ロスターCSVから取得）
- レーダー（8軸）
  - スピード
  - 加速
  - 減速
  - 再加速
  - ジャンプ
  - パワー
  - 安定性
  - バランス・左右差
- 選手サマリー
  - Home画面の8カテゴリ（スピード/加速/減速/再加速/ジャンプ/パワー/安定性/バランス・左右差）
  - 各カテゴリのスコア + 差分

**UI表記（主要ラベル）**
- ソート
- 絞り込み
- 選手
- ポジション

**操作**
- 左の選手行タップで対象選手を切り替え
- フィルタ/ソートを切り替え
- デフォルトは背番号昇順

**状態**
- 取得前: スケルトン表示
- データ欠損: 欠損箇所は「--」表示
- 権限不足: 選手名は非表示、背番号のみ表示
- 伸長率/疲労フラグ/バッジは将来拡張のため非表示

## 5. MVPにおける表示要件（第一弾）
- CSV/JSONのインポート結果をSCR-01/02に反映できることを最優先とする
- Photon（スプリント/5-0-5）/Hawkin（CMJ）の主要KPIを表示し、将来拡張カテゴリは非表示とする
- SCR-01 は選手名ドロップダウンで表示切替できること
- SCR-03は選手リスト/レーダー/差分表示までをモックで再現する
